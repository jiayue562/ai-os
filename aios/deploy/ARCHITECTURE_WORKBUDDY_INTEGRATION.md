# AI龙龟共生伙伴操作系统 - WorkBuddy集成版架构设计

## 系统定位
基于WorkBuddy技能生态的完整AI操作系统，整合：
- **ai-os**：基础设施层（四层记忆、技能框架、工作流）
- **龙心OS**：核心调度引擎（1+5智能体架构）
- **龙脑OS**：思维模型与方法论引擎
- **龙爪OS**：专业领域执行引擎
- **五行人格心理学OS**：人格分析系统

## 核心组件映射

### 1. 调度引擎（龙心OS集成）
```
原龙心OS总智能体 → AI OS核心调度器
功能：
- 意图识别（六大需求类型）
- 场景分类（S0-S9场景矩阵）
- 引擎路由（决策树v2.0）
- 动态调整（升级/降级机制）
- 结果整合（多引擎输出融合）
- 进化沉淀（自动知行合一）

集成方式：
- 复用龙心OS的SKILL.md和references文档
- 转换为Python调度器模块
- 保持1+5架构完整性
```

### 2. 记忆系统（ai-os四层记忆）
```
SOUL层：存储信仰、文化、人格、思维模型
USER层：存储用户偏好、历史、行为、成长
TOOLS层：存储技能库、API、工作流配置
SESSION层：存储当前对话、临时数据、上下文

集成方式：
- 直接使用ai-os/scripts/memory_manager.py
- 扩展支持龙心OS场景记忆
- 添加技能调用历史记录
```

### 3. 技能调用层（WorkBuddy集成）
```
技能发现：
- 扫描WorkBuddy/skills/目录
- 读取skill.yaml配置文件
- 构建技能注册表

技能调用：
- 通过WorkBuddy CLI调用技能
- 参数传递和结果解析
- 错误处理和重试机制

技能协同：
- 龙心OS调度多个技能协同工作
- 技能间数据传递
- 结果合并和冲突解决
```

### 4. 专业领域技能映射
```
龙脑OS → 思维模型引擎
  - 9大高频本能模型
  - 20个中频模型（四大模式）
  - 71个低频模型（工具层）
  - 集成到调度器决策树

龙爪OS → 执行引擎
  - 项目和工作流管理
  - SOP标准化流程
  - 领域专业知识库

五行人格心理学OS → 人格分析引擎
  - 五行人格分析

  - 团队建设支持
```

## 数据流设计

### 用户请求处理流程
```
1. 用户输入（自然语言）
2. 上下文感知（读取对话历史、身份信息）
3. 意图识别（🔍获取信息/📖深度理解/💡创新突破/⚖️分析决策/🎯任务执行/🔄系统进化）
4. 场景分类（S0-S9场景矩阵 + 复杂度评估）
5. 引擎路由（决策树v2.0 → 选择技能组合）
6. 技能调用（通过WorkBuddy调用对应技能）
7. 结果整合（融合多技能输出）
8. 进化沉淀（自动知行合一，更新系统）
9. 用户输出（结构化结果）
```

### 记忆读写流程
```
写入流程：
1. 每次对话生成完整日志
2. 根据重要性评分存储到对应层级
3. 重要记忆自动归档到Obsidian

读取流程：
1. 根据当前场景检索相关记忆
2. 优先读取高频重要记忆
3. 提供上下文给调度决策
```

## 接口设计

### 自然语言接口
```
输入格式：
- 直接自然语言描述需求
- 支持上下文延续
- 支持文件附件

输出格式：
- 结构化回答（Markdown格式）
- 可执行建议
- 后续问题引导
```

### WorkBuddy技能调用接口
```
调用协议：
skill_name: 技能名称
parameters: JSON格式参数
context: 当前会话上下文
priority: 执行优先级

响应格式：
status: success/error
data: 技能执行结果
metadata: 执行元数据
```

### 配置接口
```
系统配置：
- 记忆存储路径
- Obsidian集成配置
- 技能白名单/黑名单
- 调度策略参数

用户配置：
- 个人偏好设置
- 常用技能快捷方式
- 通知和提醒设置
```
